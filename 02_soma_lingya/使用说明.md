## 1. 概述

Soma LingYa（Soma LingYa）是基于灵芽通道（LingYa Channel）的神经网络参数高效微调方法，完全替代LoRA的低秩分解范式。

### 核心特性

| 特性 | 灵芽 | LoRA |
|------|------|------|
| 数学原理 | 门控调制 Y = XW + λ·Gate(X)·X | 低秩分解 ΔW = BA |
| 推理开销 | **零开销**（可融合） | 需要同时加载两套权重 |
| 参数效率 | **<0.01%** 原始参数 | 0.1%-1% 原始参数 |
| 表达能力 | 更灵活的门控机制 | 受低秩假设限制 |
| 训练稳定性 | 天然收敛稳定 | 需要谨慎的学习率调度 |

### 核心创新

1. **数学框架革新**：不依赖低秩分解假设，采用全新的门控调制框架
2. **推理零开销**：参数可完全融合进原始权重
3. **delta clamp修复**：确保P矩阵范数稳定，避免训练发散

## 2. 与LoRA的本质区别

### 2.1 数学原理对比

**LoRA (Low-Rank Adaptation)**:
```
ΔW = B @ A
W' = W + α · ΔW
```

**Soma LingYa (LingYa Channel)**:
```
W = I + R @ P · α
```

其中：
- R ∈ R^{d_out × rank}: 脚手架矩阵（冻结的正交基）
- P ∈ R^{rank × d_in}: 生长矩阵（从零开始训练）

### 2.2 参数效率对比

| 模型规模 | LoRA参数 | 灵芽参数 | 节省比例 |
|----------|----------|----------|----------|
| 512维度, r=8 | 8,388 | 4,194 | **50%** |
| 1024维度, r=8 | 16,776 | 8,388 | **50%** |
| 2048维度, r=8 | 33,552 | 16,776 | **50%** |

### 2.3 推理开销对比

| 方案 | 推理时需要 | 额外内存 | 推理延迟 |
|------|-----------|----------|----------|
| LoRA | W + ΔW两套权重 | +0.1%~1% | +5%~15% |
| **灵芽** | **仅融合权重W** | **几乎为零** | **零开销** |

## 4. 快速开始

### 4.1 基本用法

```python
import mlx.core as mx
from soma_lingya import LingYaChannel, ChannelType

# 创建灵芽通道
channel = LingYaChannel(
    d_in=512,
    d_out=512,
    rank=8,
    channel_type=ChannelType.BRANCH,
    growth_scale=1.0
)

# 前向传播
x = mx.random.normal((2, 32, 512))
output = channel(x)

# 计算梯度
grad_output = mx.random.normal((2, 32, 512))
gradients = channel.compute_gradients(grad_output, x)

# 生长更新
channel.grow(gradients, lr=0.01)
```

### 4.2 融合推理

```python
# 训练完成后固化
channel.freeze()
W_fused = channel.W_frozen  # 融合后的权重

# 推理时直接使用融合权重，无额外开销
output = x @ W_fused.T
```

### 4.3 灵芽块（多通道）

```python
from soma_lingya import LingYaBlock

# 创建灵芽块（包含多个通道）
block = LingYaBlock(
    d_model=512,
    num_channels=4,
    rank=8
)

# 前向传播
x = mx.random.normal((2, 32, 512))
output = block(x)
```

## 5. API参考

### 5.1 LingYaChannel

**`__init__(d_in, d_out, rank, channel_type, growth_scale, max_growth)`**

初始化灵芽通道。

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| d_in | int | - | 输入维度 |
| d_out | int | - | 输出维度 |
| rank | int | 8 | 灵芽秩 |
| channel_type | ChannelType | BRANCH | 通道类型 |
| growth_scale | float | 1.0 | 生长尺度因子 |
| max_growth | float | 5.0 | P矩阵最大范数（delta clamp） |

**`__call__(x, record_usage)`**

前向传播。

**`compute_gradients(grad_output, x)`**

计算P矩阵梯度。

**`grow(gradients, lr)`**

执行一次生长更新。

**`freeze()`**

固化通道，融合权重。

### 5.2 ChannelType

通道类型枚举：
- `ROOT`: 根通道，保留原始特征
- `BRANCH`: 枝通道，特征组合
- `LEAF`: 叶通道，细节捕捉

### 5.3 LingYaBlock

灵芽块，包含多个灵芽通道。

## 6. 训练流程

### 6.1 完整训练示例

```python
import mlx.core as mx
from soma_lingya import LingYaChannel

# 1. 创建通道
channel = LingYaChannel(d_in=512, d_out=512, rank=8)

# 2. 训练循环
for step in range(1000):
    # 前向传播
    x = get_batch()  # 获取训练数据
    output = channel(x)
    
    # 计算损失
    loss = compute_loss(output, target)
    
    # 反向传播
    grad_output = compute_grad(loss, output)
    gradients = channel.compute_gradients(grad_output, x)
    
    # 生长更新
    channel.grow(gradients, lr=0.01)

# 3. 固化
channel.freeze()

# 4. 推理
output = x @ channel.W_frozen.T  # 零开销
```

### 6.2 消融实验 - Delta Clamp修复

**问题**: 旧版本灵芽未实现delta clamp，导致P矩阵范数可能过大，PPL恶化-1.2%

**修复**: 添加max_growth参数限制P范数

```python
# 修复后的代码
channel = LingYaChannel(
    d_in=512,
    d_out=512,
    rank=8,
    max_growth=5.0  # 限制P范数
)
```
