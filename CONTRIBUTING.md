# 贡献指南

感谢你对 Dalin L 的贡献！本文档将帮助你了解如何为项目做出贡献。

## 📋 贡献流程

### 1. Fork 仓库

点击 GitHub 页面上的 "Fork" 按钮，创建你的副本。

### 2. 克隆仓库

```bash
git clone https://github.com/YOUR-USERNAME/dalin-l.git
cd dalin-l
```

### 3. 创建分支

```bash
git checkout -b feature/your-feature-name
```

### 4. 开发功能

遵循项目的代码规范和审查流程。

### 5. 提交更改

```bash
git add .
git commit -m "feat: add your feature"
```

### 6. 推送代码

```bash
git push origin feature/your-feature-name
```

### 7. 创建 Pull Request

在 GitHub 上创建 Pull Request，填写详细的描述。

## 🔍 审查流程

所有贡献必须通过五方辩论共识引擎审查：

| 审查方 | 重点 |
|--------|------|
| **Alpha** | 技术正确性 |
| **Beta** | Agent 友好度 |
| **豆包** | 中文支持 |
| **GPT** | 理论正确性 |
| **混元** | 工程实现 |
| **元宝** | 用户体验 |

## 📝 代码规范

### Rust 代码

- 使用 `rustfmt` 格式化代码
- 使用 `clippy` 检查代码
- 所有公共函数必须有文档注释
- 测试覆盖率 > 90%

### Dalin L 代码

- 使用 `dalan fmt` 格式化代码
- 所有函数必须有文档注释
- 使用中文注释

## 🐛 报告 Bug

如果你发现 Bug，请创建一个 Issue，包含：

- 问题描述
- 复现步骤
- 预期行为
- 实际行为
- 环境信息

## 💡 提出新功能

如果你想提出新功能，请创建一个 Issue，包含：

- 功能描述
- 使用场景
- 预期行为
- 替代方案

## 🤝 行为准则

### 我们的承诺

为了营造一个开放和友好的环境，我们承诺：

- 使用包容性的语言
- 尊重不同的观点和经验
- 优雅地接受建设性批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心

### 不可接受的行为

- 使用性化的语言或图像
- 人身攻击
- 恶意挑衅
- 公开或私下骚扰
- 未经许可发布他人信息
- 其他不道德或不专业的行为
