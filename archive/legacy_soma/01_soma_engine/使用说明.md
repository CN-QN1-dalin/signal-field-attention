## 1. 概述

Soma Engine（Soma Engine）是基于信号场（Signal Field）注意力机制的神经网络推理加速系统，。

### 核心特性

| 特性 | 数值 |
|------|------|
| 单层解码加速 | 最高 **4.16x** |
| 平均加速比 | **2.15x** |
| 内存压缩 | 最高 **248x** |
| 参数开销 | 仅 **8.1KB** |

### 技术原理

Soma Engine采用双通道注意力机制：
- **近场通道**：使用固定容量的Ring KV Buffer存储最近k个token
- **远场通道**：使用信号场状态向量S提供全局压缩信息

通过这种设计，将传统O(n²)的注意力计算降低为O(k·n)，其中k=16为固定谐振模式数量。

## 4. 快速开始

### 基础用法

```python
import mlx.core as mx
from soma_engine import SignalFieldLayer, EngineConfig

# 创建引擎
config = EngineConfig(
    dims=128,
    num_heads=4,
    k=16,
    gamma=0.98,
    alpha=0.1
)
engine = SignalFieldLayer(config)

# 准备输入 [batch, seq, dims]
x = mx.random.normal((1, 32, 128))

# Prefill阶段：编码输入序列
output, field_state, ring_buffer = engine.prefill(x)

# 单步解码：O(1)复杂度
x_new = mx.random.normal((1, 1, 128))
output_new, new_field_state, new_ring_buffer = engine.decode_step(
    x_new, field_state, ring_buffer
)
```

### 使用工厂函数

```python
from soma_engine import create_soma_engine

# 快速创建引擎
engine = create_soma_engine(
    dims=512,
    num_heads=8,
    k=16
)
```

## 5. API参考

### 5.1 EngineConfig

引擎配置数据类。

```python
@dataclass
class EngineConfig:
    dims: int = 128           # 模型隐藏维度
    num_heads: int = 4        # 注意力头数量
    num_kv_heads: int = 2     # 键值头数量（GQA）
    head_dim: int = 32        # 每个头的维度
    k: int = 16               # 谐振模式数量
    gamma: float = 0.98        # 衰减因子
    alpha: float = 0.1         # 远场注意力权重
```

### 5.2 SignalFieldLayer

信号场注意力层。

#### 方法

**`__init__(config: EngineConfig)`**
- 初始化信号场层

**`full_forward(x: mx.array) -> mx.array`**
- 全量前向传播（参考实现）
- 参数：`x` - 输入张量 [batch, seq, dims]
- 返回：输出张量 [batch, seq, dims]

**`prefill(x: mx.array) -> Tuple[mx.array, mx.array, RingKVBuffer]`**
- Prefill阶段：编码输入并构建推理状态
- 参数：`x` - 输入张量 [batch, seq, dims]
- 返回：`(output, field_state, ring_buffer)`

**`decode_step(x_new, field_state, ring_buffer) -> Tuple[mx.array, mx.array, RingKVBuffer]`**
- 单步解码：O(1)复杂度的增量推理
- 参数：`x_new` - 新输入，`field_state` - 信号场状态，`ring_buffer` - 环形缓冲区
- 返回：`(output, new_field_state, new_ring_buffer)`

### 5.3 RingKVBuffer

环形键值缓冲区。

#### 方法

**`write(k_vec, v_vec)`**
- 写入键值对

**`read() -> Tuple[mx.array, mx.array]`**
- 按时间顺序读取所有有效键值对

## 常见问题

### Q: 为什么prefill和full_forward输出完全一致？
A: Soma Engine的prefill方法与full_forward使用完全相同的计算逻辑，只是通过Ring KV Buffer和field_state维护了增量状态。

### Q: 如何选择合适的k值？
A: k=16是经过大量实验验证的最优值。过小的k会丢失信息，过大的k会增加计算开销。

### Q: Soma Engine适用于哪些场景？
A: 特别适用于长序列推理场景（如文档理解、代码生成）、边缘设备部署（内存受限）、实时对话系统（低延迟要求）。
