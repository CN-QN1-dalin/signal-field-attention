## 一、整体评价

### 1.1 核心优势

1. **创新方向有价值**：用信号场（EMA + Ring Buffer）替代自注意力是一个真实的技术问题，O(k)内存 vs O(n)内存的差距在长序列场景下确实有意义
2. **正确性验证扎实**：benchmark_suite.py 的实际运行结果显示 t≥1 时 Sim=1.0（跳过 t=0），这是三篇论文中最坚实的证据
3. **理论框架有层次感**：信息容量定理、信息瓶颈解释、Delta Clamp 有界性保证，比 v1.0 有显著提升
4. **消融实验覆盖面广**：10 项补充实验涵盖了策略、鲁棒性、失败案例等多个维度

### 1.2 致命弱点（必须解决）

以下问题是顶会审稿人最可能直接拒稿的原因：

1. **实验数据大量是模拟/推算的，不是真实运行得出的**
2. **SFA 在 MLX 原型上的预填充实际上比标准 Attention 慢 10×，论文声称"更快"存在误导**
3. **7B 模型只有架构验证，没有实际训练结果**
4. **"Soma Labs" 作为机构缺乏可信度**

### 论文 1：Soma Heritage（蒸馏）

#### 🔴 CRITICAL（必须修改）

**C1. 三层 PPL 数据缺乏可复现性**

论文声称：
- Layer 0: +3.07%
- Layer 11: -0.57%  
- Layer 23: -10.57%

**问题**：
- 这三个数字是如何得到的？是一次训练还是多次平均？
- 随机种子是多少？
- 训练 800 步就收敛了吗？PPL 曲线是否稳定？
- Layer 23 的 -10.57% 意味着 SFA 在深层"超越"了标准注意力，这在理论上说不通（SFA 是近似，不可能比精确注意力更好），除非存在某种正则化效应。论文完全没有解释这个反直觉现象。

**审稿人会问**：为什么深层反而更好？这是过拟合还是正则化？能否给出训练曲线？

**建议**：
- 明确标注这些数字的来源（单次运行 vs 平均）
- 给出随机种子
- 对 Layer 23 的超越给出合理解释（可能是 SFA 的近似起到了隐式正则化作用，需要引用相关文献）
- 给出 PPL 收敛曲线图

**C2. 下游任务准确率是推算的，不是实测的**

论文表 6 明确写了"基于经验回归系数估算"。这意味着 LAMBADA +1.76% 这个数字没有任何实际实验支撑。

**审稿人会问**：如果下游任务没有实际评测，论文如何证明蒸馏后的模型在真实任务上有价值？

**建议**：
- 要么删除下游任务部分，要么实际跑一次 LAMBADA/PIQA 评测
- 如果算力不够，至少坦诚说明这是"基于 PPL 的理论推断"

**C3. GradNorm 实验数据不可信**

论文声称 GradNorm 将权重从 (1.0, 0.5, 0.1) 调整为 (0.86, 0.35, 0.06)。但看 `扩展实验.py` 中的实现：

```python
losses = [0.5 + step*0.001, 0.3 - step*0.0005, 0.1 + step*0.0001]
```

这些损失值是**人为构造的线性函数**，不是真实的蒸馏损失。这意味着 GradNorm 的效果是在虚构数据上验证的，没有实际意义。

**建议**：
- 要么标注这是 Toy Experiment，要么在真实蒸馏训练上运行 GradNorm 并报告真实权重变化

**C4. 跨数据集验证的数据来源不明**

PTB 的 Baseline PPL=23.5 和 SignalField PPL=22.8 是怎么来的？代码里 `cross_dataset_validation()` 函数直接返回硬编码值，没有实际运行。

#### 🟡 MAJOR（应该修改）

**M1. 信息容量定理的证明有缺陷**

定理 1 的证明：
> "要使 t=k 时保留率 ≥ 1-ε，即 γ^k ≥ 1-ε。取对数得 k ≤ log(1-ε)/log(γ)"

这里有个方向错误：应该是 k ≥ log(1-ε)/log(γ)（因为 log(γ) < 0，不等号方向反转）。论文写的是 k ≤，这是数学错误。

此外，"乘以 κ 得到保守估计"这一步完全没有数学依据。条件数如何与 EMA 衰减关联？这是一个跳跃。

**M2. 距离核逼近引理过于简单**

Lemma 1 的"证明"只是做了代换 γ = e^(-1/σ)，然后说 γ^k = e^(-k/σ)。这不是一个"引理"，只是一个代数恒等式。它没有回答关键问题：标准注意力的核 K(i,j) = exp(-|i-j|/σ) 中的 σ 应该如何从数据中学习？

**M3. 层重要性评分没有实际算法支撑**

公式 I(l) = κ(Aₗ) × ‖∇ℒₗ‖ 看起来很合理，但：
- 注意力矩阵 Aₗ 的条件数在训练过程中是动态变化的，如何实时计算？
- 梯度范数 ‖∇ℒₗ‖ 在蒸馏训练中，已冻结层的梯度为零，如何区分？
- 代码中用的是 mock 数据（`mock_attention_cond` 和 `mock_grad_norms`），没有实际计算

**M4. 渐进式替换的学习率调度缺乏理论支撑**

Proposition 2 声称 ηₗ = η₀/l^β 可以使梯度均衡，证明思路是"深层梯度方差更大"。但这个假设需要验证：
- 梯度方差真的随层数线性增长吗？
- 不同的任务/模型是否遵循同一规律？
- β=0.5 是最优的吗？还是经验调出来的？

**M5. 与现有蒸馏方法的对比不完整**

论文对比了 TinyBERT、DistilBERT、ALBERT，但没有对比：
- **ELER** (Efficient Language Model via Knowledge Distillation)
- **Reverse Distillation** (去除教师噪声的蒸馏)
- **Attention Distillation** (直接匹配注意力图)
- **Logit Distillation with Temperature Scaling**

**M6. 表 3 的 "一次性全层替换" 数据是模拟的**

one_shot_degradations 完全是 `8.0 + layer_idx * 0.5` 这样的公式生成的，没有实际实验。审稿人会质疑这个对比的有效性。

#### 🟢 MINOR（建议修改）

- N 1. 表中使用 emoji（✅🎉）不符合学术论文规范
- N 2. "Soma Labs" 作为机构名称，如果是个人研究应标注为 "Independent Researcher"
- N 3. 参考文献 [4] Sun S et al. ALBERT 不完整（缺少作者全名和会议信息）
- N 4. 中文论文应提供完整的英文摘要，目前英文摘要是翻译体，建议润色

### 论文 2：Soma Native（架构）

#### 🔴 CRITICAL（必须修改）

**C1. 核心主张与实验数据矛盾**

论文声称 Soma Native 是"从零设计的原生架构"，但：
- 所有实验都是基于 Qwen2.5-0.5B 的**注意力层替换**，不是从头训练一个新架构
- 表 9 中 7B 模型的 PPL 标注为 "TBD"，意味着没有实际训练结果
- "原生架构"的叙事与实际"模块替换"的实现之间存在巨大鸿沟

**审稿人会问**：如果 PPL 还没验证，凭什么说这是一个可行的架构？

**C2. 推理延迟数据是理论推算，不是实测**

表 8 中：
- Standard Attention 在 seq=65536 时的延迟 2880μs
- SFA 在 seq=65536 时的延迟 11.8μs

这些数据来自 `inference_latency_comparison()` 函数，使用的是**理论公式**，不是实际硬件测试。而且 11.8μs 这个数字在所有序列长度下完全不变，这在现实中是不可能的（即使 O(1) 也有常数开销的变化）。

**C3. 内存数据存在计算错误**

论文声称 7B 模型在 640K 序列下 Soma Native 仅需 462KB。但看 `benchmark_results.json` 的实际数据：

```json
"len_65536": {
    "standard_kb": 458752.0,
    "soma_kb": 115.5,
    "compression_ratio": 3971.9
}
```

这里 dims=896（0.5B 配置），不是 7B。7B 的 dims=3584，内存应该是 115.5 × (3584/896)² ≈ 1.8MB，不是 462KB。

**C4. 预填充阶段 SFA 比标准 Attention 慢 10×**

benchmark_suite.py 的实际运行结果显示：

| 序列长度 | Standard Prefill | Soma Prefill | Speedup |
|---------|-----------------|-------------|---------|
| 64 | 1.12ms | 10.19ms | **0.11x** |
| 1024 | 6.70ms | 164.54ms | **0.04x** |
| 4096 | 63.67ms | 688.46ms | **0.09x** |

SFA 在预填充阶段**比标准 Attention 慢 10×**，论文却声称"32.8% FLOPs 更少所以更快"。这是因为 MLX 是 Python 框架，循环实现的 SFA 有大量 Python 开销。论文没有区分"理论 FLOPs"和"实际运行时"的区别。

**建议**：必须在论文中明确标注"当前数据来自 Python/MLX 原型，C++/Metal 部署后才能达到理论速度"。

**C5. Homeostasis 和 GrowthTemporal 没有实验验证**

论文提出了 Homeostasis（替代 LayerNorm）和 GrowthTemporal（替代 RoPE），但：
- 这两个组件没有任何实验数据
- 没有与 RMSNorm/RoPE 的对比
- 在统一场块中的实际贡献未知

**审稿人会问**：如果核心创新组件都没有验证，论文的科学价值在哪里？

#### 🟡 MAJOR（应该修改）

**M1. 复杂度分析的表述不准确**

论文声称 SFA 的计算复杂度是 O(k·n·d)，但标准 Attention 也是 O(n²·d)。当 k 是常数时，SFA 确实是 O(n·d)，但这只在 k ≪ n 时成立。论文没有讨论 k 随 n 增长的情况。

**M2. 与 Mamba 的对比过于片面**

表 2 中 Mamba 的"不支持全局注意力交互"是一个简化。Mamba-2 已经引入了选择性注意力机制。论文应该引用最新的 Mamba-2 论文。

**M3. 参考文献不完整**

- 缺少 Linformer、Performer、BigBird 等线性注意力相关工作
- 缺少 RWKV、RetNet 等线性复杂度架构的对比

#### 🟢 MINOR

- N 1. "Soma Native" 与 "Soma Engine" 的关系不清楚（前者是架构，后者是模块？）
- N 2. 表 1 中的 Small/Medium/Large 配置没有与任何实际模型对标

### 论文 3：Soma LingYa（PEFT）

#### 🔴 CRITICAL（必须修改）

**C1. 与 LoRA 的对比不公平**

论文声称 LingYa 在相同秩下节省 50% 参数，但：
- LoRA 训练两个矩阵（A 和 B），LingYa 训练一个矩阵（P），冻结一个矩阵（R）
- 这本质上是比较"训练 2 个矩阵" vs "训练 1 个矩阵 + 1 个固定矩阵"
- 如果 LoRA 也冻结其中一个矩阵（比如只训练 A），参数量就一样了

**审稿人会问**：如果 LoRA 也冻结 B 矩阵，参数量就与 LingYa 相同了。LingYa 的真正优势是什么？

**C2. 融合推理延迟 16% 的提升数据来源不明**

表 4 中融合前 250ms vs 融合后 210ms，但：
- 没有说明测试的 batch size
- 没有说明测试的序列长度
- 没有说明是单次推理还是批量推理
- 没有给出标准差

**审稿人会问**：16% 的提升是在什么条件下测得的？能否复现？

**C3. 消融实验的数据是模拟的**

表 6 中不同通道组合的 PPL 数据（23.1, 22.8, 22.5, 22.9）没有说明是实际运行还是推算。从 `扩展实验.py` 来看，这些数字很可能是模拟的。

**C4. 没有与 QLoRA/DoRA/AdaLoRA 对比**

论文在局限性中承认了这一点，但这在 PEFT 领域是不可接受的。2026 年的 PEFT 论文如果不与 QLoRA 对比，审稿人会认为工作不完整。

#### 🟡 MAJOR

**M1. Delta Clamp 的阈值 5.0 是经验值**

论文给出了 Delta Clamp 的公式，但 τ_max=5.0 的选择依据是什么？是否有理论分析？还是调出来的？

**M2. 脚手架矩阵 R 的初始化策略缺乏理论支撑**

ROOT/BRANCH/LEAF 三种通道的命名和分类看起来很有创意，但：
- ROOT 就是单位矩阵的前 r 列，这本质上就是 LoRA 的 B=Aᵀ 特殊情况
- BRANCH 的 SVD 初始化与 AdaLoRA 的奇异值初始化类似
- LEAF 的小噪声初始化与 LoRA 的随机初始化类似

审稿人可能会说：这些"创新"只是对已有方法的重新包装。

**M3. 参数效率证明过于简单**

定理 1 的证明只是做了除法 dr/2dr=1/2。这在数学上没错，但没有回答关键问题：用一半的参数能否达到相同的表达能力？

#### 🟢 MINOR

- N 1. "灵芽通道"的命名在英文论文中建议改为 "Sprout Channel" 或 "Sprout Module"
- N 2. 参考文献 [3] 的 Rebuffi et al. 与 PEFT 关系不大，建议替换为更相关的工作
