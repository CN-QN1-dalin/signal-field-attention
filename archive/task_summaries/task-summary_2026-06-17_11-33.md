# Task Summary: 五岳逐模块源码审计

## Objective
对太初五岳开源项目的所有8个独立模块+6个论文模块进行逐文件源码级审计，生成真实架构地图，区分"真金"和"沙子"。

## 审计方法
- 逐模块阅读完整源代码（不只是论文和README）
- 对照论文声称 vs 代码实际实现
- 识别重复算法、伪实现、营销包装

## 核心发现

### 真正有价值的（2个）
1. **SFA Engine** — 环形缓冲区 + EMA场状态，cos_sim=1.000000 (t≥1)，有实测数据
2. **LingYa PEFT** — 正交基微调 ΔW=P·V，参数是LoRA一半，Gram-Schmidt正交化有数学基础

### 部分有价值的（3个）
3. **Native** — Homeostasis/GrowthTemporal是伪实现，统一场块概念有用
4. **Convergence** — 和SFA Engine高度重复，只是RingKVBuffer+EMA的包装
5. **Heritage** — 蒸馏框架代码合理但PPL数据全是模拟的

### 没有价值的（5个）
6. **华岳** — 零训练=随机权重，混合架构替换无实际意义
7. **归元** — 就是信号场EMA的变体
8. **RCA** — Performer RFF注意力，已有成熟方案
9. **Metal** — 理论估算，无实测
10. **Ultra** — 通用部署优化，和SFA无关

### 核心公式（去装饰后只剩3行）
```
F_t = γ·F_{t-1} + (1-γ)·mean(k_t)  // EMA场状态
output = softmax(Q·K_near^T)·V_near + α·F_t  // 双通道注意力
ΔW = P·V  // LingYa正交基微调
```
