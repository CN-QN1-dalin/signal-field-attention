# Task: True SFA Implementation in llama.cpp — Build Success

## Objective
Implement genuine Signal Field Attention (SFA) in llama.cpp, combining dual-channel attention (near-field ring buffer + far-field EMA), homeostatic regulation, and LingYa PEFT — replacing the previous skeleton that only used standard FLASH_ATTN_EXT.

## Key Reasoning

### Problem Identified
Previous `dalin_soma.cpp` was a skeleton: it registered the architecture correctly but used standard `build_attn()` (FLASH_ATTN_EXT) for both SFA and non-SFA branches. The SFA-specific tensors (field_state, resonance, LingYa P) were loaded but never used in the graph.

### Solution: True Dual-Channel SFA
Implemented the actual SFA algorithm from the papers:

1. **Near-field channel**: Standard ring buffer attention via ISWA (sliding window)
   - QKV projection → RoPE → FlashAttention on kv_swa
   - Provides precise local attention on recent tokens

2. **Far-field channel**: EMA aggregation of all history
   - `F_t = γ·F_{t-1} + (1-γ)·mean(K_t)` where γ = 1 - β
   - Field state persisted as per-layer tensor [n_kv_heads, head_dim]
   - Projected through output weight for far-field contribution

3. **Fusion**: `attn_output = near_field + α × far_field_proj`
   - α controlled by `soma.alpha` (default 0.1)

4. **Homeostasis**: Adaptive per-dimension regulation
   - `norm × (1 + homeostasis_reg × 0.01)`
   - Vector loaded from GGUF tensor [n_embd]

5. **LingYa PEFT**: Zero-initialized growth matrix
   - `W = I + R@P·α`, where R=identity, P starts at zero
   - Applied as residual to FFN output

### Technical Challenges Solved
- **Nested class access**: `graph` is nested in `llama_model_dalin_soma`, can't directly access `layers_data`. Fixed by casting `model` to `const llama_model_dalin_soma&`.
- **Missing `build_arch_graph`**: Declared but not defined, causing linker error. Added definition.
- **No `ggml_add_scalar`**: Used `ggml_add(ctx, tensor, ggml_new_f32(ctx, value))` instead.
- **No `ggml_mean`**: Used `ggml_sum_rows` + scaling for mean computation.
- **Removed dead helpers**: Deleted unused `build_decay_table`, `build_far_field`, `build_lingya_ffn` functions.
