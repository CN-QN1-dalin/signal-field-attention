# Task Summary: SFA 精华集成计划

## Objective
系统梳理太初五岳项目五个核心模块的实际代码能力，提取可移植到 llama.cpp 的精华部分，制定具体集成方案。

## Key Reasoning

### 1. 源码审计发现
对 14 个模块的源码进行了全面阅读，确认了三个独立实现层：

| 层级 | 实际实现 | 与论文差距 |
|------|---------|-----------|
| **SOMA X 大脑** | 预测编码 + surprise 检测 | 完全不同（不是注意力机制） |
| **开源五岳 MLX 原型** | 真正实现了 SFA 环形缓冲 + EMA 场状态 | ✅ 核心算法已验证 |
| **llama.cpp 集成** | 架构骨架正确，SFA 核心未实现 | 骨架，核心未实现 |

### 2. 各模块精华提取

**SFA Engine (SFA_Metal.cpp/.metal)** — 最有价值：
- ✅ Ring Buffer KV 管理（固定大小 O(k)）
- ✅ 双通道注意力融合（near_field + α·far_field）
- ✅ EMA 远场聚合（F_t = γ·F_{t-1} + (1-γ)·k_t）
- ✅ 6 个 Metal GPU 内核（near_field_attn, ema_update, dual_path_fusion, qkv_project, output_project, ring_write）
- ✅ CPU 参考实现 + correctness 验证

**Soma LingYa (源代码.py)** — PEFT 零开销：
- ✅ 正交脚手架 R（SVD 分解）+ 零初始化生长 P
- ✅ 三种通道类型（ROOT/BRANCH/LEAF）
- ✅ Delta Clamp 正则化 + 固化机制（推理零开销）
- ✅ 比 LoRA 少 50% 参数

**Soma Convergence (源代码.py)** — O(1) 增量推理：
- ✅ RingKVBuffer 类（完整环形缓冲区抽象）
- ✅ GaussianDecayTable（预计算高斯衰减）
- ✅ Prefill/Decode 一致性保证
- ✅ 状态序列化/反序列化

**Soma Native (源代码.py)** — 原生架构思想：
- ⚠️ SignalFieldLayer（概念性，有简化）
- ✅ Homeostasis 稳态调节（替代 LayerNorm）
- ✅ GrowthTemporal 生长时序（可学习位置编码）

**Soma Heritage (源代码.py)** — 蒸馏框架：
- ✅ 可学习压缩查询（k 个向量替代完整 KV）
- ✅ 三层蒸馏损失（特征 + 逻辑 + 状态一致性）
- ✅ 渐进式替换策略

**归元v2 (guiyuan.py)** — 纯压缩：
- ✅ 高斯衰减累积压缩
- ✅ Prefill/Decode 一致性验证

### 3. llama.cpp 当前状态
- ✅ 架构注册正确（LLM_ARCH_DALIN_SOMA）
- ✅ KV 键映射完整
- ❌ Ring Buffer KV 管理未实现
- ❌ EMA 场状态未实现
- ❌ 双通道注意力未实现
- ❌ LingYa/Heritage/Homeostasis 均未实现
- **本质：当前是"带 Soma 元数据标签的标准 Transformer"**

## Conclusions

### 集成方案（4 阶段）
1. **Phase 1 P0**：核心 SFA — Ring Buffer KV Cache + 双通道注意力（来自 SFA Engine + Convergence）
2. **Phase 2 P1**：LingYa PEFT — 正交脚手架 + 零初始化生长（来自 Soma LingYa）
3. **Phase 3 P2**：Heritage 可学习压缩（来自 Soma Heritage）
4. **Phase 4 P3**：Homeostasis + GrowthTemporal（来自 Soma Native）

### 关键技术决策
- Ring Buffer 应在 KV cache 层实现（扩展 `llama_kv_cache_iswa`）
- Metal 内核注册为 ggml 自定义算子
- LingYa 固化后将权重融合进原始层，推理零开销
- 双通道注意力用 ggml 现有算子组合实现（不强制新增 ggml 算子）

### 验收标准
- C++ SFA 与 Python 参考实现 cosine > 0.9999
- 64K 序列 KV 内存 ≤ 1MB
- Metal decode ≥ CPU 的 3x
- LingYa 固化后推理延迟增加 = 0
