# Task Summary: llama.cpp 源码深度分析 — SFA 集成方案审视

## Objective
从 llama.cpp 维护者/贡献者视角，深度审视 Dalin Soma 全量集成到 llama.cpp 的方案，确保每个判断都有源码行号佐证。

## Key Reasoning

### 1. 架构验证
- llama.cpp 已有 140+ 架构（`src/llama-arch.h:23` enum llm_arch），SFA 只需新增一个 `LLM_ARCH_DALIN_SOMA`
- 非标准注意力有成熟先例：`GGML_OP_SSM_SCAN` (Mamba), `GGML_OP_RWKV_WKV6` (RWKV), `GGML_OP_GATED_DELTA_NET` (Gated Linear Attention)
- `ggml_backend_cpu_device_supports_op()` default 返回 true，新增 op 不会破坏 CPU 兼容性

### 2. 五种集成方案评估
- **方案 A（新增 GGML OP + 新架构）**：最彻底，仿 Mamba/RWKV 路径，~15 个文件修改，上游友好
- **方案 B（MAP_CUSTOM1 原型）**：快速但局限（只支持 1 输入，无反向传播），适合短期验证
- **方案 C（替换 KV Cache 为 RingBuffer）**：最小侵入，改 `llama-kv-cache.cpp` 即可，所有架构自动受益
- **方案 D（Backend 插件）**：过于隔离，不推荐
- **方案 E（混合策略）**：Phase 0 方案 C 验证 → Phase 1 方案 A 完整集成 → Phase 2 上游 PR

### 3. 精确修改清单
为方案 A 提供了精确到行号的修改清单，覆盖：
- `ggml/include/ggml.h` — 新增 GGML_OP_SFA_ATTN
- `ggml/src/ggml-cpu/ops.cpp` — CPU 实现
- `ggml/src/ggml-metal/ggml-metal-ops.cpp` — Metal 实现
- `src/llama-arch.h/cpp` — 新架构注册
- `src/models/dalin-soma.cpp` — 新建模型文件
- `src/llama-kv-cache.cpp` — 可选 RingBuffer 替换

### 4. llama.cpp 视角的风险
- 反向传播缺失（Phase 1 只做推理）
- GGUF 格式需独立 architecture name
- Metal 着色器需复用现有 pipeline 机制
- RingBuffer 并发安全需参照 llama_kv_cells_vec

## Conclusions
- **方案 A + C 混合策略**是最务实路径：先增量验证（C），再全面替换（A）
- llama.cpp 社区已有大量非标准架构 PR 通过（Mamba/RWKV/SSM），SFA 集成有充分先例
- 下一步：Phase 0 原型验证——替换 KV cache 为 SFA RingBuffer
